# SideMemo プライバシーポリシー

最終更新日: 2026-04-27

SideMemo（以下「本拡張機能」）は、ユーザーのプライバシーを最重要視して設計されています。本ポリシーでは、本拡張機能がどのようなデータを扱い、どのように扱わないかを明記します。

## 1. 収集する情報

本拡張機能は **いかなる個人情報も収集しません**。

- 利用統計、テレメトリ、クラッシュレポートを送信しません。
- 解析ツールやサードパーティ製の SDK を一切組み込んでいません。
- アカウントの作成・サインインを必要としません。

## 2. 端末内に保存されるデータ

本拡張機能は、ユーザーが入力したメモ・タグ・設定を、ブラウザの **IndexedDB**（あなたの端末上のローカルストレージ）にのみ保存します。

具体的には以下のデータが保存されます。

- メモ本文（Markdown）、タイトル（本文 1 行目から自動生成）、作成日時、更新日時
- ユーザーが作成したタグ（名前・色）
- メモに添付したページ参照（URL、ページタイトル、ファビコン URL、添付日時）
- 設定値（レイアウトモード、テーマ、フォントサイズ、ツールバー表示、自動保存遅延など）
- 選択テキスト挿入の一時的な中継データ（`chrome.storage.session`、ブラウザを閉じると消去）

これらのデータは外部に送信されることはなく、ユーザー自身の端末から離れません。

## 3. 外部通信

本拡張機能は、サーバーや第三者サービスへの通信を一切行いません。

- 自動更新は Chrome ウェブストア / Edge アドオンの仕組みで行われます（拡張機能本体のコードからネットワーク通信は行いません）。
- ユーザーがメモに貼り付けたリンクをクリックして開く場合、ブラウザの通常のページ遷移が発生します。これは本拡張機能による送信ではありません。
- 「+ ページ添付」機能は **`chrome.tabs` 権限のみ** を使用し、現在のタブの URL・タイトル・ファビコン URL のみを取得します。ページの本文や DOM、Cookie、履歴にはアクセスしません。

## 4. 権限の利用範囲

| 権限 | 利用目的 |
|------|---------|
| `sidePanel` | サイドパネル UI として動作するために必要 |
| `storage` | ユーザー設定の保存、選択テキスト挿入の一時中継 |
| `tabs` | 「ページ添付」のために現在のタブの URL・タイトル・ファビコンを読み取る |
| `contextMenus` | 右クリックメニュー項目（パネルを開く / 選択テキストを挿入）の登録 |
| `downloads` | エクスポート JSON ファイルの保存ダイアログ表示 |

ホスト権限（`host_permissions`）は一切要求しません。これは、本拡張機能が任意の Web ページの内容を読み書きしないことを意味します。

## 5. データの削除

ユーザーは以下の方法で本拡張機能のデータを完全に削除できます。

1. ブラウザの「拡張機能を削除」を実行する（IndexedDB を含むすべての保存データが削除されます）
2. ブラウザのサイトデータ削除機能で本拡張機能のストレージをクリアする

事前にエクスポートしておけば、後から再インストールしてインポートで復元できます。

## 6. 子どもの利用について

本拡張機能は特定の年齢層を対象として設計されたものではなく、いかなる個人情報も収集しないため、子どもの利用についても追加のリスクはありません。

## 7. 本ポリシーの変更

本ポリシーを変更する場合は、ストア掲載ページとリポジトリの当該ファイルを更新します。重大な変更がある場合は、拡張機能のバージョン番号と変更履歴を通じて告知します。
