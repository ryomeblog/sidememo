# リリース手順チェックリスト

ストア掲載までの確認手順をまとめたものです。新バージョンを上げるたびに上から順にチェックしてください。

## 1. コード側

- [ ] `package.json` の `version` を更新（ストア提出版は `manifest.json` の `version` と必ず揃える）
- [ ] `manifest.json` の `version` を更新
- [ ] `npm run lint` がクリーン
- [ ] `npm run build` がクリーン
- [ ] `dist/` を読み込んで実機チェック（チェックリストは下記）

## 2. 実機チェック

ブラウザに `dist/` を読み込んで以下を確認:

- [ ] 拡張機能アイコンクリックでサイドパネルが開く
- [ ] `Ctrl+Shift+M` で開閉できる
- [ ] 任意のページで右クリック →「SideMemo を開く」「選択テキストを挿入」両方が動く
- [ ] 初回起動でウェルカムメモが入っている
- [ ] 新規メモ作成 → 1 行目 `# タイトル` がメモ一覧に反映される
- [ ] Milkdown の整形（見出し / 太字 / 斜体 / 取消線 / リスト / チェックボックス / インラインコード / コードブロック / リンク / 区切り線）
- [ ] ツールバー各ボタンが対応する書式を適用する
- [ ] タグ作成・削除・編集
- [ ] タグフィルタ複数選択（OR）
- [ ] あいまい検索（タイトル・本文横断）
- [ ] パネル幅を 350px / 500px に変えて自動レイアウト切替
- [ ] テーマ切替（自動 / ライト / ダーク）
- [ ] フォントサイズ変更
- [ ] 自動保存遅延設定（250 / 500 / 1000ms）
- [ ] ページ添付・デタッチ・チップクリックでタブ open
- [ ] 選択テキスト挿入（パネル開いていない状態 / 開いている状態の両方）
- [ ] エクスポート → JSON ファイル保存
- [ ] インポート（merge / replace 両方、replace は確認ダイアログ）
- [ ] 設定画面のストレージ使用量表示・永続化要求
- [ ] 削除確認ダイアログ（メモ・タグ）

## 3. ストア用素材

- [ ] アイコン 128×128 が用意できている（`public/icons/icon128.png`）
- [ ] スクリーンショット 3〜5 枚を `doc/store/screenshots/` に保存（`doc/store/screenshots.md` 参照）
- [ ] プロモーションタイル 440×280（任意）
- [ ] マーキー画像 1400×560（任意）

## 4. 配布用パッケージ

```bash
npm run build
npm run pack
```

`release/sidememo-<version>.zip` が生成されます。これを Chrome ウェブストア / Edge アドオンの提出フォームにアップロードします。

## 5. ストア提出

### Chrome ウェブストア

- [ ] 開発者アカウントの登録（5 USD の登録料、初回のみ）
- [ ] 「新しいアイテム」→ zip をアップロード
- [ ] 説明文（短い / 詳細）を `doc/store/description-ja.md` から貼り付け
- [ ] スクリーンショット、アイコンをアップロード
- [ ] カテゴリ: 「生産性」
- [ ] 言語: 「日本語」
- [ ] プライバシーポリシー URL を入力（GitHub Pages / リポジトリの raw URL でも可）
- [ ] 権限の justification を `doc/store/permissions-justification.md` から貼る
- [ ] Data Usage で「I do not collect or use user data」を選択
- [ ] 公開範囲（公開 / 限定公開 / 非公開）を選択
- [ ] 提出 → 審査（数日〜数週間）

### Edge アドオン

- [ ] パートナーセンターでアカウント登録（無料）
- [ ] 同じ zip と素材で提出
- [ ] Microsoft 側の追加質問（ターゲットユーザー、年齢区分など）に回答

## 6. 公開後

- [ ] ストアのリンクを README に追加
- [ ] 利用者からのフィードバック導線（Issue リンク）を確認
