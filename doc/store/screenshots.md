# スクリーンショット撮影ガイド

Chrome ウェブストア / Edge アドオンの掲載ページに必要な画像のリストと、SideMemo で撮るべき構図のメモです。

## 必要な画像（Chrome ウェブストア）

| 種類 | サイズ | 必須 | 推奨枚数 |
|------|------|------|---------|
| プロモーションタイル（Small） | 440 × 280 | 任意 | 1 |
| マーキー画像（Marquee） | 1400 × 560 | 任意 | 1 |
| スクリーンショット | 1280 × 800 または 640 × 400 | **必須** | 3〜5 枚 |
| アプリアイコン | 128 × 128 | 既に用意済 (`public/icons/icon128.png`) | - |

> Edge アドオンも同等。スクリーンショットは 1280 × 800 推奨。

## 撮影前の準備

1. `npm run build` 後、`dist/` を Chrome に「パッケージ化されていない拡張機能」として読み込む
2. ブラウザのウィンドウサイズを 1280 × 800 程度に揃える
3. テーマは「ライト」と「ダーク」両方撮りたい（最低でもライトは必須）
4. シードのウェルカムメモは消し、見栄えの良いダミーメモを 5〜10 件入れておく
   - 例: 「読書メモ」「リサーチ：◯◯」「アイデア：新機能」など
   - タグは Inbox / Idea / Work をそのまま使用
5. パネル幅はデフォルト（おおよそ 400px 以上）にして 2 ペインで写す

## 撮影する構図（5 枚）

### 1. メイン（2 ペイン・ライトテーマ）

- ファイル名: `01-main-light.png`
- 構図: 検索バー、タグフィルタバー、メモ一覧、エディタが全部入る
- メモ本文には見出し（H1/H2）、太字、リスト、コードブロックなど整形要素を含めて Milkdown の WYSIWYG が伝わるようにする
- 画面下に「ローカル保存・外部送信なし」の訴求テキストをオーバーレイしても良い

### 2. ダークテーマ

- ファイル名: `02-main-dark.png`
- 構図: 同じく 2 ペインだが、設定でテーマをダークに切り替えた状態
- 別の編集中メモを表示（タイトルだけでなく中身が見える状態に）

### 3. ページ添付 + 選択テキスト挿入

- ファイル名: `03-page-attach.png`
- 構図: 任意の Web ページ（例: 公開ドキュメント）を左に表示、SideMemo パネル右側にそのページが「ページ添付」されたチップ付きで表示
- もしくは右クリックメニュー「選択テキストを SideMemo に挿入」が開いた瞬間
- 「ページ内容には触れない（URL とタイトルのみ）」が伝わると良い

### 4. 設定画面（タグ管理 + レイアウト）

- ファイル名: `04-settings.png`
- 構図: 設定画面でレイアウトモードのカード選択と、タグ管理の編集状態が両方見える
- 上部のレイアウトカード（自動 / 2 ペイン / 1 ペイン）が選ばれている様子

### 5. 1 ペインモード（狭幅）

- ファイル名: `05-narrow.png`
- 構図: パネル幅を 350px ぐらいに狭めて、エディタタブで開かれている状態
- 「← 戻る」ボタンが見える
- レスポンシブが伝わる

## オーバーレイ用のキャッチコピー候補

- 「ブラウザの隅で、Markdown メモ。」
- 「書きながら整形。データはあなたの端末に。」
- 「サイドパネルの Markdown ノート。外部送信ゼロ。」
- 「タブを離れずに、調べてメモして整理する。」

## 保存場所

撮影した PNG は `doc/store/screenshots/` に保存してください（このディレクトリは git にコミットしておくと再申請時に使い回せます）。

## ストア提出時のサイズ調整

- 高 DPI で撮影してから 1280 × 800 にリサイズすると荒くなりません
- PNG で保存（Chrome ウェブストアは PNG/JPG 可、PNG 推奨）
- ファイルサイズは 1MB 以内目安
